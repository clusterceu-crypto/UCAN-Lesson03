# UCAN Lesson 03 — Patch 1.3.4

## Призначення

Контрольований патч уточнює сторінку **«Перед тим як перейти до практики»** після інтеграції конкретних міських кейсів у Patch 1.3.3.

Головна логіка сторінки:

1. конкретні міста й підтверджені дії;
2. чітке розділення факту, можливого управлінського застосування та переносного уроку;
3. окремі орієнтири для поглиблення;
4. окремий блок відеолекцій курсу «Циркулярна економіка»;
5. короткий місток від кейсів до власного управлінського питання;
6. компактне розкриття інформації через `details/summary`.

## Замінити файли

- `index.html`
- `css/style.css`
- `js/script.js`

Файл `js/lesson03-ai-prompts.js` включено без функціональних змін для повного контрольованого комплекту.

## Що змінено

### Доказовість міських кейсів

У кожній картці тепер окремо подано:

- проблему;
- що підтверджує офіційне джерело;
- які дані доступні;
- можливе управлінське застосування;
- що може використати громада;
- офіційне джерело.

Формулювання не приписують місту результатів, яких офіційне джерело прямо не підтверджує.

### Компактне представлення

- картка Львова відкрита за замовчуванням;
- картки Роттердама й Амстердама залишаються згорнутими;
- у кожному `summary` додано коротку формулу управлінської цінності;
- після кейсів додано питання-перехід до ситуації власної громади.

### Відео

- Лекція 3-1 залишається основним тематичним відео;
- Лекція 1-5 позначена як додаткова й подає ширшу інфраструктурну логіку.

### Збереження практичного результату у PDF

- learner-facing назву кнопки змінено з «Друк / PDF» на **«Завантажити PDF»**;
- перед відкриттям системного діалогу учасник отримує вказівку обрати «Зберегти як PDF»;
- збережено браузерний print-to-PDF механізм без зовнішніх бібліотек і передавання даних назовні;
- отриманий PDF можна зберігати, надсилати або друкувати пізніше.

### Підсумковий тест

Шість запитань збережено, але тест став прикладнішим:

- чотири питання побудовані як короткі управлінські ситуації;
- неправильні варіанти стали правдоподібнішими;
- перевіряється застосування логіки «рішення → дані → відповідальний → користь», а не лише запам’ятовування визначень;
- правильні відповіді та Test Gate залишилися технічно сумісними з попередньою версією.

## Вибір міст у наступних заняттях

У чинному курсі з **26 занять** наступні заняття **не повинні автоматично повторювати Львів, Роттердам або Амстердам**.

Кейс добирається за такими пріоритетами:

1. найкраща відповідність темі конкретного заняття;
2. офіційне джерело та достатня доказовість;
3. конкретне місто, дія й управлінський урок;
4. переносність для українських громад;
5. різноманіття географії, масштабів і типів рішень у межах курсу.

Повторне використання міста допустиме лише тоді, коли воно демонструє іншу, окремо підтверджену практику, яка є справді найкращою для теми.

## Не змінено

- теоретичні тексти сторінок 1–6;
- 12-сторінкову архітектуру;
- самоперевірку, практичну форму та Portfolio;
- learner-facing AI prompts;
- кількість тестових запитань, правильні відповіді та Test Gate;
- localStorage keys;
- затверджені Visual Assets;
- чотири офіційні ресурси для поглиблення;
- два відеопосилання.
